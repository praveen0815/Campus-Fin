🎯 HOW TO USE
───────────────────────────────────────────────────────────────────────────────

ADMIN - CREATE SLOTS:
1. Go to Admin Dashboard → Manage Slots
2. Select: Venue, Date, Session (Morning or Evening)
3. Review: Preview shows all hourly slots
4. Click: "Create Slots for Selected Session"
5. Result: All 5-7 slots created instantly
6. Verify: See slots in "Created Slots" table

STUDENT - BOOK SLOT:
1. Go to Student Dashboard → Book Slot
2. Step 1: Choose Sport (list of sports)
3. Step 2: Choose Venue (filtered by sport)
4. Step 3: Select Date (date picker)
5. Step 4: Choose Slot
   - See Morning (6-11 AM) and Evening (4-11 PM) groups
   - Each slot shows time range + "1 Hr" badge
   - Click "Book Now" on available slot
6. Confirm: See success toast and status update

⚙️ TECHNICAL SUMMARY
───────────────────────────────────────────────────────────────────────────────

Architecture:
React + TypeScript → Service Layer → Supabase PostgreSQL
         (Tailwind CSS for styling)

Session Management:
SESSION_SLOT_WINDOWS = {
  morning: [06:00→07:00, 07:00→08:00, ..., 10:00→11:00],  // 5 slots
  evening: [16:00→17:00, 17:00→18:00, ..., 22:00→23:00]   // 7 slots
}

Bulk Operations:
# Instead of individual inserts:
await supabase.from('slots').insert([slot1, slot2, slot3, slot4, slot5])

Real-Time Sync:
supabase
  .channel(`slots-${venueId}-${date}`)
  .on('postgres_changes', { event: '*', table: 'slots' }, loadSlots)
  .subscribe()

Safety Mechanisms:
1. Before creating slots: Check if already exist
2. Before booking: Check if already booked
3. On booking: Update slot.status to 'booked'
4. Real-time: Other users see update instantly
